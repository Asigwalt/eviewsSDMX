Download SDMX data in EViews

This API allows you to retrieve datasets or series in EViews from different providers by their ID.

Currently, the app is supporting Insee, ECB & Eurostat data natively.

Requests to Quandl, BLS and FRED are also supported.

Alternatively, SDMX requests to others providers can be made here.

I. How does it work?

The app is transforming SDMX flows from providers website in an HTML table that can be read by EViews. The app provides you with a stable URL that you can use in your EViews code.

Downloading data to EViews is done in two steps:

  • get the urlof the html table containing the data.
  • import the data into EViews.

Importing data

  • from Insee, ECB, Eurostat (provider): http://sdmx.herokuapp.com/PROVIDER/RESOURCE/ID_RESOURCEwhere the resource can be either dataset or series.
  • from Quandl: http://sdmx.herokuapp.com/quandl/API_KEY/ID_RESOURCE
  • from the BLS: http://sdmx.herokuapp.com/bls/API_KEY/ID_RESOURCE
  • from an SDMX url http:sdmx.herokuapp.com/req?url=SDMX_URL

Example of EViews code:

%url = "http://sdmx.herokuapp.com/insee/dataset/IPCH-2015-FR-COICOP?freq=M"
wfopen(wf=ipch,type="html") %url colhead=2 namepos=first
With an url you can retrieve:
  • a timeseries
  • a dataset
  • multiple timeseries (solely on the Insee website)

A. Get a Timeseries

Use the ids of the timeseries: http://sdmx.herokuapp.com/series/idbank

You can filter the results and limit the number of observations by adding the following filters:
  • fixing the number of observations with the filter: lastNObservations or firstNObservations
  • fixing the starting period with the filter: startPeriod
  • fixing the ending period with the filter: endPeriod
Example:%url = "http://sdmx.herokuapp.com/insee/series/000436387?startPeriod=2010"

B. Get multiple Timeseries (only supported for Insee data)

You could add multiple idbanks to your request by separating each idbank by a '+': http://sdmx.herokuapp.com/series/idbank1+idbank2+idbank3

Example: http://sdmx.herokuapp.com/series/001762151+001762152+001762153?startPeriod=2010

Be cautious ! Previous filters still work but you have to ensure that all the timeseries share the same time period. Otherwise, some values can be missing.

C. Get a Dataset

To get a dataset, you should know its id. An exhaustive list is available:

  • for Insee: http://sdmx.herokuapp.com/insee/dataflow

  • for ECB: http://sdmx.herokuapp.com/ecb/dataflow

  • for Eurostat: http://sdmx.herokuapp.com/eurostat/dataflow

Then, the url for the dataset data is: http://sdmx.herokuapp.com/service/dataset/id_dataset

Some datasets have multiple dimensions. For instance, CPI Inflation dataset (IPC-2015-COICOP) contains both monthly and annual data. Thus, you might need to know the dimensions of a dataset before retrieving it.
An exhaustive dimensions lists can be found here: http://sdmx.herokuapp.com/insee/dataflow/id_dataset

The dimension are used to filter out the response. You must add '?nom_de_la_dimension=dimension' at the end of the URI.

When using numerous dimensions, you must separate them by '&'.

Example: http://sdmx.herokuapp.com/insee/dataset/IPCH-2015-FR-COICOP?FREQ=M

Example: http://sdmx.herokuapp.com/ecb/dataset/EXR?FREQ=M&CURRENCY_DENOM=EUR&EXR_TYPE=SP00&EXR_SUFFIX=A

Skim Data

The app allows you to skim through the available datasets and timeseries from different services. A list of all the available datasets for each provider can be found at the url http://sdmx.herokuapp.com/provider/dataflow. The lists provide with links to information on each dataset.

Example

EViews Input: 

%url = "http://sdmx.herokuapp.com/insee/series/000436387?startPeriod=2010&endPeriod=2016"
wfopen(wf=ipch,type="html") %url colhead=2 namepos=first

EViews Output:

A workfile with a two series, one series named num000436387 containing information from the timeseries with idbank 000436387 and a series Datescontaining Dates from 2010M01 to 2016M05

Known Issues

Since the app output is an html table, an obvious limitation is that a request cannot be done along multiple time dimensions. You cannot retrieve in the same request monthly and quaterly data.

Moreover :
  • Using the filter ?lastNObservations= when retrieving multiple series or a dataset is a bad idea if series do not belong to the same time period.
  • Currently, retreiving some entire datasets in one instance is not possible due to limitation on the timeout period on Heroku website. However, you can filter results in ordrer to retrieve the datasets in multiple instances.
